# あんぴノート 低コスト本番運用・収益化計画

## 1. 最初の方針

最初はスマホアプリとしてストアへ出す前に、PWAとして実利用テストを行う。

- 配布: URL共有 + ホーム画面に追加
- 対象: 身近な家族、知人、少人数のテストユーザー
- 課金: 最初は無料または手動決済で検証
- 通知: LINE公式アカウント + メール代替
- 目的: 「毎日押してくれるか」「家族が通知を信頼するか」「月額を払う価値があるか」を確認する

この段階では、App StoreやGoogle Playの審査、ストア手数料、ネイティブアプリ保守を避ける。

## 2. 低コスト構成

### 推奨構成

- フロントエンド: Next.js PWA
- 認証: Firebase Authentication メール/パスワード
- データ: Cloud Firestore
- LINE連携: LINE Messaging API
- Webhook/API: Firebase Functions または Next.js API Route
- 監視: Firebase Console、LINE Official Account Manager

### 無料枠中心で始める場合

- Firebase Authenticationはメール/パスワードなら低コストで始めやすい
- Firestoreは小規模なら無料枠で十分に検証できる
- LINE公式アカウントは無料プランから開始する
- PWA配布ならApple Developer ProgramやGoogle Play Console登録費用が不要

### 注意

Firebase Cloud Functionsを使う場合、FirebaseのBlazeプラン設定が必要になる。無料枠はあるが、請求先登録が必要。
完全に請求先登録を避けたい場合は、最初はLINE自動Webhookを使わず、メール招待と手動確認で検証する。

## 3. LINE通知コストを抑える設計

LINE通知は「毎日送る」のではなく、「異常時だけ送る」設計にする。

- 通常時: 本人がチェックインするだけで家族通知は送らない
- 期限到来: 本人へアプリ内表示またはメールでリマインド
- 猶予時間後: 家族へLINE通知
- 連続通知: 1日1回まで

これにより、LINE無料枠を長く使える。

目安:

- 100人の利用者がいて、月に5%だけ未チェックイン通知が発生する場合、LINE通知は月150件程度
- まずはLINE無料枠で検証可能

## 4. スマホアプリ化の順番

### Step 1: PWAで実利用

費用を最小化し、すぐに家族へ試してもらう。

成功条件:

- 20組以上が1週間使う
- 7日後継続率が50%以上
- 家族側から「通知があると安心」と言われる
- 月額300円なら払う意思が複数件ある

### Step 2: Androidアプリ化

AndroidはGoogle Play Console登録が一回払いのため、iOSより先に出しやすい。

方法:

- PWAをTrusted Web ActivityでAndroidアプリ化
- 既存Next.js/PWAを大きく作り直さない
- Google Playで「見守り」「安否確認」検索流入を確認

### Step 3: iOSアプリ化

iOSはApple Developer Programの年会費があるため、課金見込みが見えてから行う。

方法:

- Capacitorで既存PWAをアプリ化
- Push通知やApp Store課金は必要になった段階で追加

## 5. 収益化設計

### 無料プラン

- 本人1名
- 家族連絡先1名
- メール代替通知
- LINE通知は月数回まで

目的: 使い始めの心理的ハードルを下げる。

### 有料プラン 月額300円

- 家族連絡先3名
- LINE通知
- 通知履歴
- 確認頻度の変更
- 広告なし

まずは月額300円が現実的。家族側が払う価値を感じやすい。

### ファミリープラン 月額500円

- 見守り対象2名まで
- 家族連絡先5名
- 優先通知

夫婦で親を見守るケースや、兄弟で分担するケースに向く。

## 6. 損益の目安

最初の固定費を抑える場合:

- PWAのみ: ほぼ無料枠で開始可能
- LINE無料枠内: 追加費用なし
- Firebase無料枠内: 追加費用なし

有料化後の目安:

- 月額300円で30人: 月9,000円
- 月額300円で100人: 月30,000円
- 月額500円で100人: 月50,000円

ストア公開後は、Apple/Googleの手数料や年会費を考慮する。
ストア手数料を避けたい初期段階では、PWA上のWeb決済や銀行振込などで支払意思を検証する。

## 7. リリース前に必要な本番対応

- Firebase本番プロジェクト作成
- Firestore Security Rules反映
- LINE公式アカウント作成
- LINE Messaging APIチャネル作成
- LINE Webhook URL設定
- プライバシーポリシーの運営者情報追記
- 利用規約の運営者情報追記
- 問い合わせメールアドレス設定
- 通知不達時の免責表示をアプリ内に明記
- ログ削除・アカウント削除の導線を追加

## 8. 判断基準

スマホアプリ化は、次の条件を満たしてから行う。

- PWAで30組以上が利用
- 2週間継続率が40%以上
- 月額300円以上の支払意思が10件以上
- LINE通知が月無料枠を超えそう
- 「ホーム画面追加が難しい」という声が複数ある

この条件を満たす前にネイティブアプリ化すると、開発・審査・保守コストが先に増える。
