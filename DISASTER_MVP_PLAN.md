# 家族防災ノート MVP 設計

## 1. 画面構成案

- ホーム: 今日の防災ステータス、家族人数、最終更新日、備蓄点検、期限アラート、緊急モード導線
- 家族: 家族一覧、続柄、緊急連絡先、注意事項、最新状態、緊急連絡先
- 緊急モード: 無事、要支援、返信困難、定型文、手動位置共有、家族状況一覧
- 防災ノート: 避難場所、集合ルール、緊急時連絡手順、服薬、アレルギー、注意事項
- 備蓄チェック: 水、食料、モバイルバッテリー、常備薬、子ども用品、ペット用品、期限
- 設定: 月次点検、同期ON/OFF予定、位置共有ON/OFF、端末データ削除、法的ページ

## 2. コンポーネント構成

- `DisasterNoteApp`: Phase 1 のメインPWA UI
- `SafetyApp`: 既存の安否確認UI。削除せず残し、将来の同期や招待機能に再利用可能
- `InviteAcceptance`: 既存の家族招待承認
- `PushRegistration`: 既存のアプリ通知登録

## 3. データモデル定義

- `users`: 将来の同期用ユーザー
- `households`: 世帯単位の同期用グループ
- `household_members`: 家族、続柄、連絡先、注意事項、最新状態
- `emergency_contacts`: 学校、親族、勤務先などの緊急連絡先
- `evacuation_places`: 避難場所、集合場所、メモ
- `disaster_rules`: 連絡手順、集合ルール、家族内ルール
- `medical_notes`: 服薬、アレルギー、持病、配慮事項
- `supply_items`: 備蓄品、カテゴリ、数量、期限、点検状態
- `safety_status_logs`: 有事の状態共有ログ
- `notification_settings`: 月次点検、同期、位置共有、通知設定

Phase 1 は `localStorage` に `DisasterNoteData` として保存し、Phase 2 で Supabase または既存APIに接続する。

## 4. ルーティング設計

- `/`: 家族防災ノート本体
- `/terms`: 利用規約
- `/privacy`: プライバシーポリシー
- `/disclaimer`: 免責事項
- `/invite/[code]`: 既存の家族招待承認。Phase 2 以降で防災ノートの家族共有と統合

## 5. モバイルUI案

- 6タブのスマホ優先UI: ホーム、家族、有事、ノート、備蓄、設定
- 緊急モードは大きい3ボタンにする
- 位置共有は初期OFFで、手動入力のみ
- 重要情報はカード化し、入力欄は縦並びにする
- 有事でも読めるように、説明文より状態と行動を優先する

## 6. PWA / Offline 対応方針

- アプリシェルをService Workerでキャッシュ
- 防災ノートの最低限データは端末保存
- オフライン時も家族、連絡先、避難場所、備蓄、医療メモを閲覧可能
- Phase 2 で同期キュー、競合解決、バックアップを追加

## 7. 法的ページの文案

- 利用規約: 防災情報整理と共有補助であり、救助、医療判断、安全保証ではないことを明記
- プライバシーポリシー: 取得情報、利用目的、保存先、削除方法、家族共有範囲を明記
- 免責事項: 通信不達、位置情報の正確性、公的通報代行ではないことを明記
- 未成年は保護者同意と管理を前提とする

## 8. MVP 実装順序

Phase 1:
- ホーム
- 家族管理
- 防災ノート
- 備蓄チェック
- 緊急モード
- 法的ページ

Phase 2:
- クラウド同期
- 月次通知
- PDF出力
- 有料版制限

Phase 3:
- 多言語
- 高齢者向け簡易UI
- 家族別権限
- 災害履歴分析
